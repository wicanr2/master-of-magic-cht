# 工作魔法大帝 繁體中文化 — 版本說明

這一版做了三件事:主要畫面全部中文、修好一批老玩家回報的玩法問題、把原版設定畫面缺的選項補回來。
三個平台(Windows / macOS / Linux)的安裝包都已更新。

---

## 修好的玩法問題

這些都是玩家實際回報、玩起來會卡住或不對勁的狀況,現在都修好了:

- **走到海邊怪物巢就直接輸**
  以前帶著陸軍走進海邊的怪物據點,整支軍隊會憑空消失、不戰而敗。現在會正常開打。

- **打造神器的成本錯亂(存讀檔之後)**
  以前在打造強力神器的過程中存檔、再讀檔,成本會跑掉——兩三回合就做好、法力不夠也照做。
  現在讀檔後成本維持正確,強力神器該花多少時間就花多少。

- **點遠方海面有時無法移動**
  以前叫船去很遠的海域,或點斜角的海格,有時會發出錯誤聲、選不了,有時又正常。
  現在船能正常規劃航線過去。

- **自訂巫師打名字打到一半被踢回去**
  以前在建立自訂巫師、輸入名字時,只要按到大小寫切換鍵(Caps Lock)就會被踢回上一個畫面,
  逼你用極快的速度打完名字。現在輸入名字不會再被踢出去。

- **召喚第七個英雄沒反應**
  以前英雄已經滿六個、還想再召喚或雇用第七個時,會無聲無息地失敗,讓人摸不著頭緒。
  現在會明白提示「你沒有空間容納更多英雄,請先解僱一位」。

- **打怪拿到的裝備,當場穿也要花法力**
  以前打怪拿到裝備,連當場參戰的英雄要穿戴,都得花 20 點法力「傳送」。
  現在參戰的英雄當場穿戴免費,只有要把裝備送給遠方沒參戰的英雄時才需要那 20 點法力(這才是原版規則)。

## 看起來像 bug、其實是原版設計

有兩個玩家以為是 bug 的狀況,查證後確認是原版本來就這樣設計的,所以保持原樣——但我們把原因說清楚:

- **英雄戰死、打贏後好像又活過來?**
  其實那個英雄並沒有真的死,只是血量被打到很低、看起來像倒下;戰鬥結束後隨時間自然回血,
  所以下一場又看到他。遊戲對「真正陣亡」的處理是正確的:真的死掉的英雄不會因為打贏就復活。

- **只有白綠藍三色書,戰鬥時法術書裡卻有混亂(紅)系法術?**
  這不是法術外洩。法術研究嚴格照你擁有的書系——沒有紅書,你永遠研究不到任何混亂法術。
  你在戰鬥中看到的那些他系法術,來自**你召喚到的英雄自己會的法術**。
  原版手冊第 82 頁就寫明:英雄自帶的法術不算在巫師的法術書裡,只會在戰鬥中出現。
  所以一位混亂系英雄上陣時帶幾發混亂法術,完全符合原版規則。

## 補回的設定選項

原版的設定畫面有一整排可開關的選項,這個重製引擎原本只做了音量。這一版把其中 16 項補回來,全部中文:

- **戰鬥**:全自動戰鬥(所有戰鬥自動結算、不用親自指揮)、戰鬥中顯示當前單位的彈藥/法力/血量。
- **聲音**:音效開關、背景音樂開關。
- **畫面/操作**:法術施放動畫開關、顯示已控制魔法節點的光環、顯示視野內敵軍的移動、自動結束回合
  (所有單位行動完就自動進下一回合)、法術書按類別或按顏色排列。
- **通知**:對手施放召喚 / 城市結界 / 戰略結界 / 一般法術時提示你、回合開始顯示當回合的事件總結。
- **隨機事件**:火山、地震、好月壞月等隨機事件的開關。
- **自動建議**:回合開始時,提醒你該注意的事——例如某座城市快餓死了、金幣或法力每回合在虧、
  某座城市沒有守軍、目前沒有在研究任何法術等等。這些提醒的規則,是我們從五份以上的玩家攻略
  整理出來的(整理成 [`docs/mom-strategy-notes.md`](mom-strategy-notes.md))。

有兩個原版選項這次沒做:事件音樂(與背景音樂分開控制,實用性低)、展開式說明(只是把說明視窗改成
從點擊處放大的動畫,純美觀)。

## 萬一閃退

遊戲若不幸閃退,現在會把過程記錄到一個檔案(`crash.log`,放在系統的設定資料夾)。
回報問題時附上這個檔,我們就能看到閃退前發生了什麼、比較容易找出原因。

## 怎麼安裝

三個平台都自帶中文字型與譯文,不需另外安裝任何東西;你只要自備一份正版的《Master of Magic》遊戲資料即可
(本專案不附帶、也不散布任何版權遊戲檔)。

- **Windows**:解開 zip,雙擊「玩遊戲.bat」(或直接執行 `magic.exe`)。免任何額外的 DLL。
- **macOS (Apple Silicon)**:解開 zip,執行「Master of Magic CHT.app」。
  因為沒有付費簽章,第一次開啟可能要在「系統設定 → 隱私權與安全性」按一下允許。
- **Linux**:給 AppImage 執行權限後直接執行(`chmod +x` 後雙擊或在終端機執行)。

> 每一批改動都經過自動化測試與實機截圖確認,不是「編譯過就算數」。
> 戰鬥與數值相關的修正都用開關包住,預設保留重製原版的行為。
